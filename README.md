# hrishikeshpatil_bitsom_ba_2511159_part2_kpi_experiment

The Situation
We just wrapped up testing a new onboarding flow to see if we can get more new sign-ups to actually stick around and subscribe.
We split users into two groups:
Control: Got the old, current onboarding.
Treatment: Got the new campaign.
Now we need to figure out if we should roll this new version out to everyone. It’s a big call because it impacts product, marketing, support, and our revenue.
The Main Goal
We're looking at Paid Conversion Rate as our main winner-metric. Did the new flow get more people to pay?
Things We Need to Watch Out For
Even if conversion looks good, we can't break other things. We need to keep an eye on:
Refunds: Did we accidentally push people too hard, causing a spike in refund requests?
Support tickets: Is the new flow confusing? Are people flooding customer service with questions?
Bad revenue: Are these users just going to cancel immediately after the first month?
Sub-groups: Did this absolutely tank for specific types of users or locations?
What We Need to See Before Launching
We’ll only roll this out 100% if:
Paid conversion actually went up by a decent amount.
Our support and refund numbers didn't spike.
